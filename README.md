# 케이스 발표 자동화 Flow

## 목적

이 폴더는 케이스 발표를 만들 때마다 반복해서 불러오는 자동화 기준 문서입니다.

목표는 사용자가 케이스 정보만 주더라도 아래 과정을 일관되게 진행하는 것입니다.

1. 케이스 핵심 파악
2. 발표 주제 후보와 강조 질문 후보 제안
3. 사용자가 주제/질문 선택
4. 기존 PPT의 슬라이드 14 이후 구조에 맞춰 문헌고찰 flow 구성
5. 슬라이드별 bullet 작성
6. 슬라이드별 심화 설명 작성
7. 슬라이드별 citation 검색어 생성
8. PubMed 기반 논문 후보 검색
9. 논문 권위성/관련성/최신성 기준으로 최종 논문 선정
10. PPT 초안 또는 발표용 문헌고찰 초안 작성

## 사용 방법

새 케이스를 만들 때 사용자는 아래처럼 요청하면 됩니다.

```text
case_presentation_flow 폴더의 자동화 flow대로 진행해줘.

케이스:
...

수술/치료 포인트:
...

추가로 강조하고 싶은 내용:
없으면 네가 먼저 후보를 추천해줘.
```

## 문서 구성

- `01-case-input-template.md`: 케이스 입력 양식
- `02-topic-question-selection.md`: 발표 주제와 강조 질문 후보 선정 방식
- `03-slide14-review-workflow.md`: 기존 PPT 슬라이드 14 이후에 맞춘 문헌고찰 구성
- `04-citation-search-automation.md`: 슬라이드별 citation 검색 자동화
- `05-paper-selection-rubric.md`: 논문 선정 점수화 기준
- `06-output-format.md`: 최종 초안 출력 형식

## 기본 원칙

- 케이스만 주어진 경우, 바로 슬라이드를 만들지 않습니다.
- 먼저 발표 주제 후보 3-5개와 강조 질문 후보를 제안합니다.
- 사용자가 선택한 뒤에 bullet 작성과 논문 검색을 진행합니다.
- 논문 검색 과정에서 PubMed API 등 네트워크 접근이 필요하면 사용자에게 승인 요청을 보냅니다.
- 승인된 네트워크 접근은 해당 케이스의 논문 검색 작업 범위에서만 사용합니다.
